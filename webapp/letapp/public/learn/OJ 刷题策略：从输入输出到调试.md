# OJ 刷题策略：从输入输出到调试

## 课程目标

帮助你更快适应在线评测（Online Judge，简称 OJ）环境，减少因为输入输出、边界判断或格式错误导致的失分。无论你是刚接触算法竞赛，还是在准备编程考试，掌握正确的刷题策略都能事半功倍。

> **什么是 OJ？** OJ 是一个在线裁判系统，你提交代码后，系统会自动用多组测试数据运行你的程序，检查输出是否和预期一致。常见的 OJ 平台有：LeetCode、牛客网、洛谷、Codeforces、AtCoder 等。

## 推荐内容

### 输入输出规范

OJ 系统的输入输出有严格的格式要求，这是最容易失分的地方，也是最容易避免的。

#### 理解题目样例
每道题都会给出"输入样例"和"输出样例"，你需要仔细阅读它们来理解：
1. **输入格式**：有几行？每行包含什么？
2. **输出格式**：输出顺序、分隔符、是否要换行？
3. **边界情况**：样例通常比较简单，你还需要自己思考各种边界情况

#### 多组输入的处理方式

很多 OJ 题目会包含多组测试数据，常见的有以下几种形式：

**方式一：先给测试组数**
```python
# 第一行输入一个整数 T，表示有 T 组测试数据
T = int(input())
for _ in range(T):
    n = int(input())
    # 处理每组数据
    print(result)
```

**方式二：读到文件末尾（EOF）**
```python
# 不知道有多少组，一直读到输入结束
import sys
for line in sys.stdin:
    line = line.strip()
    if line == "":
        continue
    # 处理数据
    print(result)
```

**方式三：每行多个数据**
```python
# 一行包含多个整数
a, b, c = map(int, input().split())

# 或者读成一整个列表
arr = list(map(int, input().split()))
```

#### 常见输入输出陷阱

```python
# 1. 忘记去除换行符
name = input().strip()  # .strip() 去除首尾空白

# 2. 输出格式不对 —— 注意空格、换行、大小写
# 要求输出 "Case 1: 5"，注意大小写和冒号后的空格
print(f"Case {case_num}: {result}")

# 3. 浮点数精度控制
print(f"{pi:.2f}")  # 保留两位小数
print(f"{pi:.6f}")  # 保留六位小数

# 4. 输出列表元素
arr = [1, 2, 3, 4, 5]
print(" ".join(map(str, arr)))  # 输出 "1 2 3 4 5"
```

### 调试方法

调试是编程中最重要的技能之一。即使是有经验的程序员，也会写出有 bug 的代码。关键在于如何快速定位和修复问题。

#### 打印关键变量
最简单也最有效的调试方式就是在代码中添加 `print` 语句，输出关键变量的值：

```python
def binary_search(arr, target):
    left, right = 0, len(arr) - 1
    while left <= right:
        mid = (left + right) // 2
        print(f"left={left}, right={right}, mid={mid}, arr[mid]={arr[mid]}")  # 调试信息
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return -1
```

**调试技巧**：
- 在循环中打印循环变量，确认循环执行了预期的次数
- 在条件判断处打印，确认程序走了哪个分支
- 在函数开始和结束处打印，确认函数被正确调用
- 打印数组或列表的当前状态，观察数据变化

#### 分阶段验证逻辑
不要等到写完整个程序再测试，应该"边写边测"：

```python
# 第一阶段：验证输入读取是否正确
data = list(map(int, input().split()))
print("读取的数据:", data)  # 验证

# 第二阶段：验证核心逻辑
def solve(arr):
    # 假设这个函数做了某种计算
    result = sum(arr)
    print("中间结果:", result)  # 验证
    return result

# 第三阶段：验证输出
ans = solve(data)
print(f"最终答案: {ans}")
```

#### 先过样例再测边界
1. **先通过题目给出的样例**：确保基本逻辑正确
2. **再测小规模数据**：手动计算一遍，和程序输出对比
3. **再测边界情况**：
   - 最小值：n = 1 或空数组
   - 最大值：n = 10^5 或 10^9
   - 重复值：所有元素相同
   - 有序/无序：数组已经有序、完全逆序
   - 特殊字符：包含空格、空行

```python
# 自己构造测试数据
def test():
    # 测试用例 1：基本样例
    assert solve([1, 2, 3, 4, 5]) == 15
    
    # 测试用例 2：只有一个元素
    assert solve([100]) == 100
    
    # 测试用例 3：包含负数
    assert solve([-1, 0, 1]) == 0
    
    # 测试用例 4：全部相同
    assert solve([5, 5, 5]) == 15
    
    print("所有测试通过！")
```

### 常见错误

了解常见的错误类型，可以让你在遇到问题时更快地定位原因。

#### 1. 数组越界
这是最容易犯的错误之一，尤其是在使用循环时：

```python
arr = [1, 2, 3, 4, 5]
# 错误：索引从 0 开始，最大索引是 len(arr)-1
for i in range(len(arr) + 1):  # 这里多了一次循环
    print(arr[i])  # 最后一次会报 IndexError

# 正确做法
for i in range(len(arr)):
    print(arr[i])

# 或者直接用 for-each
for val in arr:
    print(val)
```

#### 2. 死循环
循环条件写错导致循环永远不会结束：

```python
# 错误：忘记更新循环变量
i = 0
while i < 10:
    print(i)
    # 忘了写 i += 1，i 永远是 0

# 错误：循环条件永远为真
while True:
    # 忘了加 break 条件
    pass
```

**避免方法**：
- 确保循环变量在每次迭代中都在向终止条件靠近
- 使用 `for` 循环代替 `while` 循环（更安全）
- 在开发时设置最大迭代次数作为保险

#### 3. 精度和类型问题
```python
# 整数除法 —— Python 3 中 / 得到浮点数
print(5 / 2)   # 2.5
print(5 // 2)  # 2（整除）

# 浮点数精度问题
print(0.1 + 0.2)  # 0.30000000000000004（不是精确的 0.3！）

# 比较浮点数时不要用 ==
# 错误做法
if 0.1 + 0.2 == 0.3:  # False！
    
# 正确做法
epsilon = 1e-9
if abs((0.1 + 0.2) - 0.3) < epsilon:
    print("相等")

# 类型转换陷阱
print("5" + 3)   # TypeError！
print(int("5") + 3)  # 8

# 输入默认是字符串
s = input()  # 用户输入 "5"
print(s * 2)  # "55"（字符串重复）
print(int(s) * 2)  # 10（数字运算）
```

#### 4. 其他常见错误

**内存超限（MLE）**：
- 创建了太大的数组，例如 `[0] * 10**8`
- 递归层数过深导致栈溢出

**时间超限（TLE）**：
- 使用了时间复杂度太高的算法（如嵌套循环过多）
- Python 中建议避免在循环中使用 `+` 拼接字符串，用列表代替

```python
# 低效做法
result = ""
for i in range(10000):
    result += str(i)  # 每次都会创建新字符串

# 高效做法
parts = []
for i in range(10000):
    parts.append(str(i))
result = "".join(parts)
```

**格式错误（PE）**：
- 多打了空格或少打了空格
- 大小写不对
- 行末多了一个空行
- 没按要求输出换行

## 练习建议

1. **为每道题自己补 3 到 5 组边界测试数据**，再提交到评测系统。可以先跑一遍自己的测试数据，确认无误后再提交
2. **建立自己的代码模板库**：把常用的输入输出模板、排序、二分查找等代码保存下来，下次直接复用
3. **时间管理**：如果一道题想了 30 分钟还没有思路，可以先看题解或讨论，理解思路后再自己写一遍
4. **错题整理**：把做错的题和原因记录下来，定期复习。特别是那些"差一点就对了"的错误，往往是进步的关键
5. **从易到难**：不要一开始就挑战难题，从简单的题开始建立信心，逐步提高难度。LeetCode 上可以按"通过率从高到低"排序来选题
6. **多语言对比**：如果时间允许，可以尝试用多种语言解同一道题，这能帮你更深入地理解语言特性和算法本质
